rathole lets you expose services running on devices behind NAT (like a home server, NAS, or Raspberry Pi) to the Internet through a server with a public IP — similar to frp and ngrok, but with significantly higher throughput and lower resource usage.

## Why rathole?

rathole is built for performance and simplicity. It handles high connection volumes with minimal memory and CPU overhead — making it ideal for resource-constrained devices like routers and embedded systems.

## How it works

rathole runs on two sides: a **server** (public IP) and a **client** (behind NAT). The client connects to the server and establishes tunnels for each service. Visitors connect to the server's exposed port, and traffic is forwarded through the tunnel to the client's local service.

<Steps>
  <Step title="Deploy a server">
    Run rathole in server mode on any machine with a public IP address
  </Step>
  <Step title="Configure services">
    Define which ports to expose on the server for each service
  </Step>
  <Step title="Connect a client">
    Run rathole in client mode on your NAT device, pointing to each local service
  </Step>
  <Step title="Traffic flows">
    Visitors connect to the server port — traffic is securely forwarded to your local service
  </Step>
</Steps>
